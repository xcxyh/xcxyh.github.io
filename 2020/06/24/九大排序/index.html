九大排序

发表于 2020-06-24 更新于 2020-09-11

九大基本排序原理，比较，复杂度，是否稳定，Java实现

[TOC]

以下介绍按照重要程度来的。不会介绍具体的实现过程，重点在于记录每种排序的特点，复杂度，稳定性，区别和使用场景。具体的实现过程使用Java代码描述。

稳定性： 判断一个排序算法是否稳定，即 排序前后 数组中两个相同元素的相对位置不发生改变，就是稳定的排序算法。

例 ：[4,3’,5,3’’,6,8] ====> [3’,3’’,4,5,6,8] 即 两个相同的3 排序前后 它们的前后位置 是一致的，即为稳定。

1. 快速排序

算法描述:任取待排序中的某一元素作为基准值,按照该基准值,将待排序集合分割成两个子序列,左边的序列比基准值小,右边的比基准值大.然后对左右子序列重复上述步骤.直到排完为止。

稳定性：不稳定。 因为在最后一次 arr[low] 与 arr[i] 交换时，如果相同的话会导致顺序变化

最坏情况复杂度：整个数组是逆序的。[8,7,6,5,4,3,2,1]， 每次划分只得到一个比上一次划分少一个记录的子序列 ， 此时它的复杂度为 O(n^2)。计算过程：
$$
T = n -1 + n -2 + n-3 …. + 1 = n(n - 1) / 2 = O(N^2)
$$
平均情况复杂度：
$$
T(n) = n + 2T(n / 2) = n + 2( n/ 2 + 2T(n/4)) = n + 2( n/ 2 + 2(n/4 + 2T(n/8)) = …
$$

$$
T(n) = n + 2T(n/2) = 2n + 4(n/4) = 3n + 8T(n/8) =…= kn + 2^kT(n/2^k)
$$

$$
又当 n / 2^k = 1 时， k = logN ,将 k = logN 带入上式：
$$

$$
T(N) = NlogN + NT(1) = O(NlogN)
$$
使用场景：是最快的通用排序算法，大多数使用情况下，是最佳选择。

2. 堆排序

堆排序的基本思想是：将待排序序列构造成一个大顶堆，此时，整个序列的最大值就是堆顶的根节点。将其与末尾元素进行交换， 此时末尾就为最大值。然后将剩余n-1个元素重新构造成一个堆， 这样会得到n个元素的次小值。如此反复执行，便能得到一个有序序列了。

堆 这里指的是 二叉堆，有大顶堆和小顶堆之分，其具体含义是，若是大顶堆，则 根节点的 值 大于等于它的 左右子树的值， 这个性质具有递归性。每个结点的左子树和右子树都是一个二叉堆（都是最大堆或最小堆）。
与 二叉树不同，二叉堆 是通过数组的方式实现的，所以我们可以利用它的性质来对数组进行排序。
具体过程是：
1首先将原数组构造成为一个二叉堆，这里我们默认构造大顶堆，那么它堆顶元素就是最大的。
2然后将 堆顶 ，即数组中的第一个位置的元素和 数组末尾元素 交换。
3然后对剩余的 0 ~ n -1 的部分进行调整，使之重新满足 大顶堆的性质。
4然后 重复 2 3 两步，直到整个数组有序。

最开始构造二叉堆时，可以从最后一个非叶子节点 的位置开始，反向遍历构造 （ (n / 2 - 1) ~ 0）

堆排序时间复杂度：O(NlogN)
初始化建堆过程时间：O(n)：（从最后一个非叶子节点开始）， k 为 堆的高度 k = logN
总的时间计算为：s = (2^( i - 1 )) * ( k - i )；其中 i 表示第几层，2^(i - 1) 表示该层上有多少个元素，(k - i) 表示子树上要比较的次数.
$$
S = 2^(k-2) * 1 + 2^(k-3)2…..+2^(k-2)+2^(0)(k-1) ===> S = 2^k -k -1 代入k : S = N - logN -1
$$

更改堆元素后重建堆时间：O(nlogn):
首先，将一个元素调整到二叉堆中合适的位置，其时间复杂度为 这个 二叉堆的高度 即 logN
然后 我们需要处理整个 数组 中的N 个元素。
合起来 他的时间复杂度为 O(NlogN)

堆排序是不稳定的。因为在堆性质维护的时候，可能导致相同的数据位置发生变换。

3. 归并排序

归并排序（MERGE-SORT）是建立在归并操作上的一种有效的排序算法,

  • 该算法是采用分治法（Divide and Conquer）的一个非常典型的应用。
  • 将已有序的子序列合并，得到完全有序的序列；即先使每个子序列有序，再使子序列段间有序。
  • 若将两个有序表合并成一个有序表，称为二路归并。

时间复杂度 O(N*logN) ；空间复杂度 O(N)

稳定性：稳定。

归并排序并不是一种原地排序，因为需要额外申请空间来充当临时容器。

可以利用该排序 求解逆序对问题。剑指 Offer 51. 数组中的逆序对

4. 冒泡排序

算法描述:比较两个相邻的元素,如果升序的话,前面的比后面的大,就交换,这样一轮下来,就会找到这组数据中最大的元素,然后抛开这个元素继续重复上述步骤.直到排完为止.

时间复杂度：O(n^2)

稳定性：稳定（如果判断相等时，不进行元素的交换）

可以使用一个标志位来进行简单的优化。

5. 选择排序

选择排序: 先找出最小的元素放到新数组中， 然后更新原数组，即去掉原数组中的这个最小元素。

时间复杂度 O(n^2)

空间复杂度 O(n)

稳定性：不稳定。

6. 插入排序

直接插入排序的基本操作就是将一个数据插入到已经排好序的有序数据中，从而得到一个新的、个数加一的有序数据， 算法适用于少量数据的排序。

时间复杂度为O(n^2)。

稳定性：稳定。

插入排序在原本升序排升序,原本逆序排逆序,元素集合接近有序时,直接插入排序效率较高.

7. 希尔排序

希尔排序(Shell’s Sort)是插入排序的一种又称“缩小增量排序”（Diminishing Increment Sort），是直接插入排序算法的一种更高效的改进版本。该方法因D.L.Shell于1959年提出而得名。
